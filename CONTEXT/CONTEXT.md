# Media Agent — Domain Language

The shared glossary for the Media Agent pipeline (Pivot.6: AI-generated Tech/AI news Shorts). This file is a glossary only — no implementation detail. Module responsibilities live in `agents.md`; the phase log lives alongside this file in `CONTEXT/`.

## Language

### Content units

**Topic**:
A tech/AI news item ingested from an RSS feed, awaiting scripting.
_Avoid_: story, article, item.

**Script**:
The narration + per-shot prompts + title an LLM writes from one **Topic**.
_Avoid_: prompt, copy.

**Shot**:
One AI-generated ~4-second video segment. A **Clip** is four **Shots** stitched in order.
_Avoid_: clip-part, segment.

**Clip**:
The finished ~16-second vertical Short — the unit that gets uploaded. Each **Clip** has exactly one **Content kind**.
_Avoid_: video, short (lowercase), render.

**Content kind**:
What a **Clip** is made of: `ai_generated` (current, Pivot.6) or `sourced` (legacy movie-clip). Gates uploader templating and disclosure.
_Avoid_: clip type, format.

**Shot kind**:
Each **Shot** is tagged `real_image` (sourced still + Ken Burns motion) or `ai_video` (Kling transition/atmosphere). A hybrid **Clip** mixes ~2 of each.
_Avoid_: shot type, media type.

**Real-image shot**:
A **Shot** built from a licensed/web-sourced still of a product, logo, or object — never a living person — with Ken Burns motion applied.
_Avoid_: stock photo, slideshow frame.

**AI-video shot**:
A **Shot** generated by Kling for abstract/transitional b-roll between real-image shots.
_Avoid_: AI clip, generated segment.

**Stitch**:
The act of combining the four **Shots** in order into one **Clip** — concatenation plus optional crossfade between adjacent **Shots**.
_Avoid_: merge, join, render.

**Shot normalization**:
Conforming every **Shot** to one canonical video format before they are **Stitched**, so heterogeneous **Shots** — e.g. a **Real-image shot** and an **AI-video shot**, which are produced at different resolutions and frame rates — can be combined. Without it, the **Stitch** fails.
_Avoid_: transcode, conform, scaling.

**Synthetic-person frame**:
A **Shot** that depicts an AI-generated human. Constrained by the no-living-individuals rule: a **Shot** prompt must never name a real person (narration may, factual reporting is fine).
_Avoid_: deepfake, fake person.

### Ship lifecycle

**First live ship**:
The first upload of a new **Content kind**. Governed by the two-gate sign-off (see `docs/adr/0001`). Distinct from steady-state daily uploads.

**Mechanics-validation ship**:
A **First live ship** whose purpose is to prove the pipeline end-to-end (upload → disclosure → Content ID → cost reconciliation), not to showcase content quality. Held to "compliant + not embarrassing," not portfolio quality.
_Contrast_: a quality ship, judged on viewer-facing polish.

**Ship gate**:
The T+1h check that the immediate upload path succeeded (disclosure visible, no Content ID claim, cost reconciled). Passing it unblocks the next slice. Marked `[~]`.

**Stability gate**:
The T+48h check that the **Clip** stayed live and clean. Marked `[x]`. Runs concurrently; does not block forward work.

## Relationships

- A **Topic** produces one **Script**.
- A **Script** produces one **Clip** and its four **Shots**.
- A **Clip** is produced by **Stitching** its four **Shots**; a hybrid mix of **Shot kinds** requires **Shot normalization** first.
- A **Clip** has exactly one **Content kind**; only `ai_generated` carries the synthetic-media disclosure.
- A **First live ship** passes the **Ship gate** before the **Stability gate**.

## Example dialogue

> **Dev:** "The narration names Corti's CEO — does that trip the no-living-individuals rule?"
> **Domain expert:** "No. The rule is about **Shot** prompts, not narration. Narration can state facts about real people. What it can't do is generate a **Synthetic-person frame** *from a prompt that names them* — that's a fake person under a real name."
> **Dev:** "So this clip is fine because the prompt said 'a CEO on a video call', not 'Andreas Cleve'?"
> **Domain expert:** "Right — and because it's a **Mechanics-validation ship**, a generic synthetic face is acceptable as long as disclosure is set."

## Flagged ambiguities

- **"test channel" vs "channel"** — every ship so far targets the YouTube *test* channel. "the channel" without qualification means the test channel until a primary channel is named.
- Pivot.7 adds: a **real_image** `entity` must be a product/logo/object, never a living person (no sourcing identifiable individuals).
